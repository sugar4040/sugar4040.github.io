# Requirements: 謎キャン2026 回遊システム

## 1. システム概要

謎キャン2026は、カタカナの答えを数字に変換してマス目を進むHTML5ベースの謎解きゲームシステムです。

### 1.1 基本コンセプト
- カタカナで答えを入力
- カタカナを数字に変換
- 数字を使ってマス目を進む
- 次の問いに移動

### 1.2 技術要件
- HTML5/CSS3/JavaScript (ES6+)
- モバイルファーストデザイン
- タッチ操作対応
- オフライン動作可能

## 2. UI/UX要件

### 2.1 画面レイアウト

#### 2.1.1 上部エリア
- 数字入力欄（横長のテキストフィールド）
- 「マス進む」ボタン（右側に配置）

#### 2.1.2 下部エリア
- 大きな「答えが表示されるところ」エリア
- タップ可能な領域

### 2.2 五十音表UI

#### 2.2.1 表示トリガー
- 答え表示エリアをタップすると五十音表が出現

#### 2.2.2 五十音表の構成
- あ行、か行、さ行、た行、な行、は行、ま行、や行、ら行、わ行
- 濁音・半濁音対応
- 小文字（ゃ、ゅ、ょ、っ など）対応

#### 2.2.3 回答欄
- 五十音表の上に回答欄が表示される
- カタカナで入力内容が表示される

### 2.3 フィードバック表示

#### 2.3.1 正解時のフィードバック
- 画面に丸の表示（○マーク）
- 音声フィードバック（オプション）
- 答えがカタカナで表示される

#### 2.3.2 不正解時のフィードバック
- エラー表示
- 再入力を促す

## 3. 機能要件

### 3.1 カタカナ入力機能

#### 3.1.1 五十音表からの入力
- 五十音表のボタンをタップしてカタカナを入力
- 入力したカタカナが回答欄に表示される
- 濁音・半濁音の変換機能

#### 3.1.2 入力の編集
- バックスペース機能（最後の文字を削除）
- クリア機能（全ての入力をクリア）

### 3.2 答え検証機能

#### 3.2.1 答えの照合
- 入力されたカタカナと正解を照合
- 大文字・小文字の正規化
- 濁音・半濁音の正規化

#### 3.2.2 正解時の処理
- 正解フィードバックの表示
- 答えをカタカナで表示エリアに表示
- カタカナを数字変換可能な状態にする

### 3.3 カタカナ→数字変換機能

#### 3.3.1 変換ルール
- カタカナの各文字を数字に変換
- 変換テーブル：
  - ア=1, イ=2, ウ=3, エ=4, オ=5
  - カ=6, キ=7, ク=8, ケ=9, コ=10
  - （以下、五十音順に対応）

#### 3.3.2 個別文字クリック機能
- 正解後の答え表示エリアで各カタカナ文字が個別にクリック可能
- 各文字をタップするとそのカタカナ（数字ではなく）が数字入力欄に追加される
- 例：「テスト」が表示された場合
  - 「テ」をタップ → 数字入力欄に「テ」追加
  - 「ス」をタップ → 数字入力欄に「テス」になる
  - 「ト」をタップ → 数字入力欄に「テスト」になる

### 3.4 数字入力機能

#### 3.4.1 カタカナ入力機能
- カタカナ文字をタップして数字入力欄に追加
- 入力されたカタカナが数字入力欄に表示される
- 「マス進む」ボタンの「マ」「ス」部分も個別にクリック可能

#### 3.4.2 区切り機能
- +ボタンを押すと数字が区切れる
- 区切り記号（+）が表示される

#### 3.4.3 分割ボタン設計
- 「マス進む」ボタンは3つの部分に分割：
  - 「マ」（クリック可能、数字入力欄にカタカナ追加）
  - 「ス」（クリック可能、数字入力欄にカタカナ追加）
  - 「進む」（機能ボタン、次の問題に移動）

### 3.5 問い移動機能

#### 3.5.1 次の問いへの移動
- 「進む」ボタンを押すと次の問いに移動
- 今入力したものが消える
- 新しい問いが表示される

#### 3.5.2 状態のリセット
- 数字入力欄がクリアされる
- 答え表示エリアがクリアされる
- 五十音表が非表示になる

## 4. データ要件

### 4.1 問題データ構造

#### 4.1.1 問題の定義
```json
{
  "id": "問題ID",
  "question": "問題文",
  "answer": "正解（カタカナ）",
  "hint": "ヒント（オプション）"
}
```

#### 4.1.2 イベントデータ構造
```json
{
  "eventId": "イベントID",
  "eventTitle": "イベントタイトル",
  "questions": [問題の配列]
}
```

### 4.2 カタカナ→数字変換テーブル

#### 4.2.1 変換マッピング
- 五十音の各文字に対応する数字を定義
- JSON形式で管理
- 設定ファイルから読み込み可能

### 4.3 進行状態の保存

#### 4.3.1 ローカルストレージ
- 現在の問題番号
- 入力中の数字
- 解答済みの問題リスト

## 5. 非機能要件

### 5.1 パフォーマンス
- ページ読み込み時間: 2秒以内
- タップ応答時間: 100ms以内
- アニメーション: 60fps

### 5.2 互換性
- iOS Safari 14+
- Android Chrome 90+
- デスクトップブラウザ（Chrome, Firefox, Safari最新2バージョン）

### 5.3 アクセシビリティ
- タッチターゲットサイズ: 最小44x44px
- コントラスト比: 4.5:1以上
- スクリーンリーダー対応

### 5.4 日本語対応
- UTF-8エンコーディング
- 日本語フォント対応
- カタカナ入力システム

## 6. 制約事項

### 6.1 技術制約
- 外部ライブラリ依存なし（vanilla JavaScript）
- ビルドステップなし
- ES6モジュール使用

### 6.2 デバイス制約
- タッチスクリーン推奨
- 最小画面幅: 320px
- オフライン動作必須
